# State Immutabiility

Pastikan tidak mengubah state dengan menggunakan nilai dari `state hasil modifikasi`. Ini dikarenakan React mendeteksi perubahan lewat perbandingan referensi. Ketika kita modifikasi objek, referensinya sama sehingga react menganggap tidak berubah. Padahal nilainya berubah tetapi referensinya sama.

Ini akan mengakibatkan `silent mutation` dimana jika kita melakukan hal tersebut, kemudian ada render yang disebabkan state lain, kita akan mendapati render state tersebut berubah, padahal kita sudah merubah state waktu sebelumnya, tetapi baru terjadi setelah render state lain.

## Contoh Salah

```jsx
state=2
setState(state);
setState(2); //benar

state.age=4
setState(state);
setState({...state, age:4});//benar

state.push(4);
setState(state);
setState([...state, 4]); //benar
```

## Prev Param Setter

Prev param juga merupakan referensi ke obj state saat ini.

```jsx
setTodos(prev => {
  prev.push({ id: 3, text: "Belajar React" }); // ❌ mutasi langsung
  return prev; // referensi tetap sama
});
```
