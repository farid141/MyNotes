# Canvas dan Webcam

Baik. Saya akan **fokus menjelaskan arti setiap parameter dan nilai**, **tanpa langsung membahas rumus crop-nya**.
Tujuannya supaya Anda paham **apa yang dikontrol oleh tiap angka** di `drawImage`.

---

## 1. `canvas.width` dan `canvas.height`

```ts
canvas.width  = video.videoWidth;
canvas.height = video.videoHeight;
```

### Artinya

* Menentukan **ukuran bitmap internal canvas**
* BUKAN ukuran tampilan CSS
* Semua gambar yang digambar akan diskalakan ke ukuran ini

### Dampak penting

* Jika canvas diset ke ukuran **landscape**
* Semua hasil `drawImage` akan berakhir landscape
* Kalau Anda ingin output portrait → canvas **harus portrait**

---

## 2. `video.videoWidth` dan `video.videoHeight`

```ts
const videoW = video.videoWidth;
const videoH = video.videoHeight;
```

* Resolusi **frame asli webcam**
* Ini **bukan ukuran DOM**
* Nilai ini datang langsung dari perangkat kamera

Contoh umum:

```bash
1920 × 1080
1280 × 720
```

---

## 3. Parameter `drawImage` (8 parameter)

```ts
ctx.drawImage(
  video,
  sx, sy, sw, sh,
  dx, dy, dw, dh
);
```

Mari kita bahas **maknanya satu per satu**.

---

### Bagian SOURCE (yang diambil dari video)

```ts
sx, sy, sw, sh
```

Ini menentukan **area mana dari video** yang diambil. Dengan memberikan nilai beracuan dari videoHeight dan videoWidth, kita dapat melakukan cropping.

#### `sx` — source X

* Posisi horizontal mulai ambil gambar
* 0 = sisi kiri video

#### `sy` — source Y

* Posisi vertikal mulai ambil gambar
* 0 = sisi atas video

#### `sw` — source width

* Lebar area yang diambil
* Kalau lebih kecil dari `videoW` → **terjadi crop kiri/kanan**

#### `sh` — source height

* Tinggi area yang diambil
* Kalau lebih kecil dari `videoH` → **terjadi crop atas/bawah**

---

### Bagian DESTINATION (digambar ke canvas)

```ts
dx, dy, dw, dh
```

Di kode Anda:

```ts
0, 0, videoW, videoH
```

Artinya:

* Gambar diletakkan mulai dari pojok kiri atas canvas
* Diskalakan agar memenuhi area `videoW × videoH`

### Dampak

* Jika `dw/dh` ≠ `sw/sh`
  → **canvas akan melakukan scaling**
  → **potensi distorsi**
