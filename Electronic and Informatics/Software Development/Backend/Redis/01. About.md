# About redis

Redis merupakan in-memory database, memiliki kecepatan akses lebih cepat dibanding database asli (karena perlu dieksekusi).

## Pengaplikasian Redis

- Caching database query (select statement)
- Session storage, berguna untuk aplikasi dengan session-based authentication
- Rate limiting, menyimpan jumlah request dari API spesifik dengan set expiration pada request pertama masuk
- Job queues, Menggunakan LPUSH, RPOP. Job disimpan dalam JSON berisi nama dan payload. Akan di-pop ketika sudah dikerjakan.

## Manfaat caching

- optimasi request ketika cpu database utama/aplikasi eksternal terlalu padat
- kalkulasi formula yang berat, menyimpan hasil kalkulasi di redis

## Installasi

Terdapat beberapa pilhan untuk mengakses redis, baik containerized atau di host langsung.

### Configuration

Redis memiliki config yang dapat diatur, walopun bisa berjalan dengan default config. Untuk menggunakan konfigurasi, tambahkan direktori config sebagai parameter saat menjalankan server.

`redis-server dir_konfig.conf`

#### Referensi

<https://github.com/redis/redis/blob/7.0/redis.conf>

Setiap konfigurasi diubah, redis harus direstart

## Mengakses Redis

Redis bekerja dengan sistem server-client, kita dapat mengakses server dengan menggunakan `redis insight, redis cli, code, dll` sebagai client.

Untuk mengakses redis kita dapat pakai `redis-cli` yang otomatis terinstal ketika instal redis.

`redis-cli -h <host> -p <port>`

host dapat berupa `localhost` atau `url server`
