# Intro

Springboot adalah framework untuk menulis aplikasi web, ada beberapa konsep yang harus dipahami dalam menggunakan springboot

- IoC Inversion of Control: menyerahkan pembuatan object, ada IoC container sebuah fitur Spring
- Dependency Injection: Menyuntikkan dependensi dengan membuat dependensi paling dasar terlebih dahulu diikuti lainnya.

### Dependency Injection

Menyuruh SpringBoot untuk membuat object, jadi kita tinggal menggunakan saja. Obj akan terbuat di dalam container.

```java
@SpringBootApplication()
public class initializer extends SpringBootServletInitializer {
    public static void main(String[] args){
        ApplicationContext context = SpringApplication.run(SpringBootDemoApplication.class, args);
        Alien obj = context.getBean(Alien.class);
        obj.code();
    }
}

@Component
public class Alien(){

    @Autowired
    Laptop laptop;

    public void code(){
        System.out.println("Coding");
    }
}

@Component
public class Laptop(){
    public void code(){
        System.out.println("Coding");
    }
}
```

Dalam code tersebut dilakukan DI dengan injection saat `.run()` dijalankan. Terdapat beberapa annotations:

- `@Component` diletakkan diatas class, membuat objek DI, bisa diberi identifier (id), jika tidak maka id menjadi `className`
- `@Scope(Singleton/Prototype)` menentukan objek DI akan dibuat secara singleton atau Prototype
- `@Primary` menandakan objek DI akan dipilih pertama jika ada confussion
- `@Autowired` menginject `variable, setter` obj DI dengan obj DI yang lain
- `@Value(23)` menginject primitive variable obj DI dengan nilai default
- `@Qualifier(identifier)` diletakkan dibawah `@AutoWired` bisa juga sebelum constructor params
