# IoC (Inversion of Control)

Untuk bekerja dengan IoC container dapat menggunakan `ApplicationContext` atau `BeanFactory` (old version).

Dalam menggunakan IoC terdapat beberapa cara:

## XML

- buat file `src/main/resources/spring.xml`
- tambahkan wrapper utama `spring bean configuration xml`

```xml
<?xml version="1.0" encoding="UTF-8"?>
<beans xmlns="http://www.springframework.org/schema/beans"
      xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
      xsi:schemaLocation="http://www.springframework.org/schema/beans
                          http://www.springframework.org/schema/beans/spring-beans.xsd">

   <!-- Bean definitions here -->
    <bean id="alien" class="com.telusko.Alien">
        <!-- setter injection -->
        <property name="age" value="21" />
        <property name="lap" ref="lap1" />

        <!-- constructor injection -->
        <constructor-args index="0" value="21" />
        <constructor-args index="1" ref="lap1" />
    </bean>

    <bean id="lap1" class="com.telusko.Laptop"></bean>

    <bean id="alien2" class="com.telusko.Alien">
        <property name="lap" ref="lap1">
            <!-- Inner Bean -->
            <bean id="lap1" class="com.telusko.Laptop"></bean>
        </property>
    </bean>

</beans>
```

### Bean

Bean merupakan object yang dikelola IoC. Untuk mendapatkan bean, gunakan

```java
(Alien) context.getBean("alien")

// atau berikan class type agar tidak perlu type casting
context.getBean("alien", Alien.class)

// atau dapatkan bean by class langsung
context.getBean(Alien.class)
```

`id` digunakan untuk identifier
`class` merupakan direktori dimana file class berada

- java based config
- annotations

### Scope

Merupakan bagaimana obj dibuat, secara default singleton

- `Singleton` buat obj sekali dan gunakan obj yang sama setiap getBean
- `Prototype` buat obj baru setiap kali `getBean` dipanggil

### Setter Injection

Dilakukan pada konfigurasi IoC untuk secara otomatis men-set nilai melalui setter.

### Ref Attribute

Digunakan untuk menggunakan bean lain sebagai nilai dengan `referensi id` bean tersebut

### Constructor Injection

Pembuatan object IoC yang membutuhkan parameter constructor, dapat dilakukan dengan 3 cara:

- secara default by urutan penulisan `constructor-args`
- props `type`, kelemahannya jika ada tipe sama
- props `index`, paling rekomended
- props `name`, sesuai nama variabel param constructor (harus menambahkan annotasi `@ConstructorProperties({"name1", "name2"})` diatas fungsi constructor)

### Auto Wire

Tidak perlu menuliskan constructor-args ada beberapa nilai:

- `byName` mencocokkan `name variable constructor` dengan `id` bean lain
- `byType` mencocokkan `type variable constructor` dengan `class type` bean lain, akan ada compiler error ketika ada 2 bean tipe sama, bisa diatasi dengan `primary="true"` pada salah satu bean

> Priority: jika pakai `constructor-args` dan `autowire` maka `constructor-args` didahulukan

### Lazy Init

Hanya akan membuat object ketika ada yang menggunakan `lazy-init="true"`

### Inner Bean

Jika kita hanya ingin spesifik bean sebagai reference bean tertentu.
