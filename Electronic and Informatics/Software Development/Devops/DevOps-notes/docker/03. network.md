# Docker network

Network adalah sebuah jaringan internal docker yang digunakan untuk `menghubungkan container dengan container lainnya`. Berbeda dengan port yang digunakan untuk menghubungkan container dengan host.

## membuat network

- `docker network create -d bridge net_name`

## beberapa opsi driver yang umum

- `bridge` The default network driver.
- `host` Remove network isolation between the container and the Docker host.
- `none` Completely isolate a container from the host and other containers.

Kontainer yang terhubung ke bridge network yang sama dapat berkomunikasi menggunakan alamat IP internal jaringan bridge tersebut.

Kita tidak perlu tahu alamat IP internal tersebut, melainkan langsung dapat mengaksesnya dengan nama container

`<container_name>:<port>`

## Connect and Disconnect

network bisa dicopot/pasang ketika container `sudah dan akan` dibuat

### Container Akan dibuat

`docker run --network=my-net -itd --name=container3 busybox`

### Container Telah dibuat

`docker network disconnect/connect net_name cont_name`
