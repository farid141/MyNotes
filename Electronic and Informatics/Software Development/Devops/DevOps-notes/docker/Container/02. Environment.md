# Pendahuluan

Container berasal dari sebuah image, dimana sebuah image dapat berisi serangkaian eksekusi command yang dapat dijalankan ketika runtime (container start) dan build time. Akan mirip seperti `env di OS` hanya cakupannya dalam container saja.

ENV yang diberikan sebagai arg dalam perintah menjalankan docker hanya hidup dalam `satu sesi saja`. Sehingga kalau container yang sudah ada, kita perlu memberikan ENV lagi. Solusi untuk hal ini adalah dengan membuat default `ENV` pada docker image, atau menjalankan container dengan `compose`.

Untuk membuat sebuah environment variabel, kita dapat menggunakan `ENV` dan `ARG`.

```bash
docker run -e PORT=3000 myimage         # Menggunakan command arg
docker run --env-file .env myimage      # Menggunakan file sebagai env
```
