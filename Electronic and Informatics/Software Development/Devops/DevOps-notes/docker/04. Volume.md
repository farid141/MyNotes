# Mounts

Terdapat 2 jenis mount:

- volume: seperti disk yang dapat di hubungkan ke container agar dapat diakses.
- bind: link dari sistem host ke container, sistem host dapat diubah baik dari host maupun container.

## mounts dapat dilakukan dengan 2 cara

```bash
docker run --mount type=bind,src=<host-path>,dst=<container-path>
docker run --volume <host-path>:<container-path>

# atau

docker run -v /path/host:/path/container ...
docker run -v nama-volume:/path/container ...
```

## Named Volume

Kita dapat menghubungkan `named-volume/host` ke container dengan perintah `volume`. Untuk named-volume, lokasinya tersembunyi

### **Linux**

```bash
/var/lib/docker/volumes/<nama_volume>/_data/
```

Kalau kamu buka folder itu (harus pakai sudo), kamu bisa melihat file-file volumenya.

### **Windows (Docker Desktop + WSL2)**

**Named volume tidak berada di Windows drive**, tapi di dalam WSL2, di:

```bash
\\wsl$\docker-desktop-data\data\docker\volumes\<nama_volume>\_data\
```

Kamu bisa membuka ini langsung melalui File Explorer:

1. Buka File Explorer
2. Di address bar ketik:

   ```bash
   \\wsl$\docker-desktop-data\data\docker\volumes\
   ```

3. Cari nama volumenya
4. Masuk ke folder `_data`

📌 **Folder `_data` adalah isi volume sebenarnya.**

## Export Named Volume

### ✔ Cara 1 — Export volume ke file TAR

```sh
docker run --rm -v <nama_volume>:/volume -v $(pwd):/backup alpine \
    tar czf /backup/backup.tgz -C /volume .
```

Contoh:

```sh
docker run --rm -v dbdata:/volume -v $(pwd):/backup alpine \
    tar czf /backup/db_backup.tgz -C /volume .
```

---

### ✔ Cara 2 — Copy volume ke folder host

```sh
docker run --rm -v dbdata:/volume -v $(pwd)/copy:/copy alpine \
    cp -a /volume/. /copy/
```

Folder `copy` sekarang berisi data dari volume.

---

### ✔ Cara 3 — Menggunakan `docker cp` (dari container)

Jika container memakai volume:

```sh
docker cp <container_name>:/path/in/container ./folder_host
```
